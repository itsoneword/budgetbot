# Decisions (ADR-lite)

Append-only. Format: `date — decision — why — rejected alternative`. Reference from task logs where relevant.

- 2026-01-25 — Storage = PostgreSQL with repository pattern; batch-fetch then filter-in-memory in `domain/` — one/two queries per handler, business logic testable as pure functions — rejected: SQL aggregation per view (query sprawl, untestable).
- 2026-07-07 — Task tracking = file-per-task kanban in `docs/tasks/` with generated `BOARD.md`, enforced by Claude Code hooks (`scripts/tasks.py`) — deterministic updates, per-task git history, works offline — rejected: single BACKLOG.md (no per-task history, coarse hook validation); GitHub Issues (state outside the repo, network-dependent).
- 2026-07-07 — Stop hook blocks once per session then allows (sentinel file) — deterministic nudge without nagging on every turn — rejected: always-block (fires on trivial turns); advisory-only CLAUDE.md rule (forgotten after compaction).
- 2026-07-08 — AI Q&A ships as prompt-packed context (load_user_session + domain/filters summary into the LLM prompt), not text-to-SQL — per-user data is tiny and the model must not touch the DB — rejected: LLM-generated SQL (injection risk, needless complexity).
- 2026-07-08 — LLM access via provider-agnostic client in infrastructure/llm/: subscription OAuth now, OpenRouter later via config — owner already pays for a subscription and wants test runs before committing to a paid API — rejected: hardcoding one provider's SDK.
- 2026-07-08 — Voice transcription via local faster-whisper, not a cloud STT — privacy and zero per-minute cost outweigh the ~1GB image growth for a personal bot — rejected: OpenAI Whisper API (second vendor + key).
- 2026-07-09: Postgres data on ./pgdata bind mount instead of named docker volume / why: survives `down -v`, visible, one backup surface / rejected: named volume (invisible, deletable by flag).
- 2026-07-09: LLM backend = claude-agent-sdk spawning host-mounted claude CLI (subscription OAuth, credentials ro) / why: zero API cost for tests, pattern proven in claude-code-telegram / rejected: raw OAuth token against API (fragile), OpenRouter now (paid; deferred to config swap).
- 2026-07-09: Docker image python 3.9 -> 3.12 / why: claude-agent-sdk requires 3.10+ / rejected: separate LLM sidecar container (more moving parts).
- 2026-07-09: /ask gated by ADMIN_USER_ID + LLM_ALLOWED_USERS env allowlist / why: LLM runs on owner's personal subscription / rejected: open to all 72 users.

- 2026-07-09: Voice/text intent routing dispatches by synthetic Update injected through application.process_update (LLM output reduced to enum + validated payload) / reuses all existing handlers, gating and conversation states; PTB v22 Message objects are immutable so mutating .text (as core.py menu_call does) crashes / rejected: calling handlers directly (show_records reads message.text -> None on voice) and duplicating the save flow.
- 2026-07-11: Recurring transactions (T-026/T-027) built code-first as an internal action API (domain + repo) consumed by both manual handlers and the AI intent router / keeps single write path, LLM only selects validated intents / rejected: letting the LLM call scheduling directly via its own tools.
- 2026-07-11: New DB tables ship as alembic revisions with T-005 landing first (T-022=0003, T-026=0004 chained). Why: two tasks both claimed 002_*.sql (collision) and raw SQL re-creates the untracked-migration drift T-005 exists to fix. Rejected: shipping raw SQL now + converting later.
- 2026-07-11: Changelog enforced as a required --changelog arg on `tasks.py done` (hook only a fail-open backstop) / why: a deterministic gate cannot be forgotten after long context, unlike a skill/instruction / rejected: skill or hook-only reminder (advisory, rots with context).
- 2026-07-11: /download CSV format = legacy 7 columns (id,timestamp,category,subcategory,amount,currency,user_id) + transaction_type appended, rendered by pure domain/export.py / migration importer ignores extras and the explicit type column frees T-015 restore from the income heuristic; same renderer to serve T-025 /admin_export / rejected: new-order schema (breaks re-import of old files).
- 2026-07-11 — Schema migrations = alembic over the async asyncpg engine (env.py rewrites postgresql:// → postgresql+asyncpg://); baseline revision replays the idempotent 001_initial_schema.sql so existing DBs upgrade in place; entrypoint runs `alembic upgrade head` on every container start — rejected: initdb.d SQL (never runs on existing volumes); adding psycopg2 just for migrations (second driver in the image).
- 2026-07-11: Bot commands live in a single registry (src/commands.py CommandSpec: name, handler-name, EN/RU desc, admin_only, in_menu) driving core.py registration, /help and set_my_commands on post_init; handler=None marks ConversationHandler entry points that must not be loop-registered / why: menu was BotFather-set and drifted from code / rejected: keeping manual BotFather list (drifts) and storing handler callables in the registry (circular import with core.py).
- 2026-07-11: AI access = DB entitlements (ai_entitlements table, NULL expires_at = perpetual, NULL revoked_at = active) checked by src/ai_access.check_ai_access with admin + LLM_ALLOWED_USERS env as fallback tier (env removed in T-023 release); repeat grant with days extends via GREATEST(expires_at, NOW()) + interval — the T-023 purchase contract; fail-closed with loud log on DB error / rejected: boolean flag on users (no source/expiry/refund semantics for the paywall).
- 2026-07-11: AI paywall = 100 Stars for a 30-day pass (env-tunable AI_ACCESS_PRICE_STARS/AI_ACCESS_DAYS; 0 days = perpetual); refund revokes all access, admin re-grants remainder manually. Rejected: one-time perpetual (LLM cost is recurring), pro-rated refunds (complexity for a rare event).
- 2026-07-11: T-035 → T-027 → T-034 run sequentially (all extend the monolithic intent prompt in domain/intent.py + voice routing); rejected intent-registry refactor for now — revisit if intents keep multiplying. T-023 can run parallel to the chain (only trivial voice.py overlap).
- 2026-07-12 — Currency API circuit breaker is in-memory per process (shared/utils/circuit_breaker.py); the DB exchange_rates table is the shared cross-process layer — rejected: DB-backed breaker state (T-008 multi-replica not started, needless writes).
- 2026-07-12: Structured logs = structlog ProcessorFormatter on the stdout handler (JSON per line); call sites stay stdlib logging.getLogger / why: structured output for docker logs + watchers with zero call-site rewrites / rejected: structlog-native rewrite of every module.
- 2026-07-12: Container healthcheck = heartbeat file (/tmp/budgetbot-heartbeat) touched by a JobQueue job after SELECT 1; compose healthcheck exec's an mtime check / why: proves event loop + JobQueue + DB in one signal, no new port / rejected: embedded HTTP health server (new port + dep for a single-process polling bot with no HTTP surface).
- 2026-07-12: Daily reminders (T-034) fire from a 5-min JobQueue sweep over DB rows with atomic claim_send (mirrors T-026 claim_run) / why: per-user JobQueue jobs are in-memory — restarts drop them and downtime silently loses fires; DB is the only state / rejected: run_daily job per user + re-registration on boot.
- 2026-07-12: User timezone (T-034) = fixed UTC offset minutes (user_configs.tz_offset_min) captured by tapping the candidate current local time / why: zero-typing one-tap UX, no tzdata dependency; DST drift of 1h accepted v1 with re-pick copy (upgrade path: swap column to IANA, only is_due input changes) / rejected: IANA zone picker (long lists/typing for a daily nudge).
- 2026-07-12: Unit testing = pytest + pytest-asyncio (pytest.ini, testpaths=tests) with hand-written fakes, gated by a GitHub Actions tests.yml on push + PRs to main; coverage printed (--cov=domain) but no --cov-fail-under in v1 / why: suite must stabilize before a threshold can gate merges, and full requirements install with pip cache is robust vs a hand-curated subset that breaks silently / rejected: coverage threshold v1, running tests inside the docker build.
- 2026-07-13: Voice corrections = context-aware re-parse (classifier re-emits the full add_* intent plus a corrects_previous flag) / why: reuses _TX_ITEM_RE and every existing guardrail unchanged / rejected: a new "correct_last" intent kind (own payload grammar, duplicated validation).
- 2026-07-13: Correcting an already-confirmed transaction = delete + re-inject corrected text through the normal save pipeline / why: re-add reuses category resolution for free / rejected: in-place UPDATE (bypasses category resolution).
- 2026-07-13: AI interaction-log retention = size-based compaction (summarize + extract key facts at ~50k tokens/user, delete raw rows, keep summaries) / why: owner wants memory persistence, size is the real constraint / rejected: 30-day purge.
- 2026-07-19: Stars paywall (T-023) grants entitlements ONLY from the successful_payment update, after an idempotent payments.record() insert (UNIQUE charge_id); invoice_payload encodes terms as sold ('ai:30'/'ai:perp') so stale invoices grant what was advertised / why: redelivered updates must not double-extend and config changes must not re-price pending invoices / rejected: granting from callback data or current env config (spoofable / wrong terms).
- 2026-07-21: Headless LLM auth = one shared `claude setup-token` in ~/.claude/service-secrets/claude-token.env (compose env_file, CLAUDE_CODE_OAUTH_TOKEN). Why: static token, no refresh chain -> no rotation races with host Claude Code; single rotation point for all services. Rejected: int_prep access-token mirror (couples services to host login health), per-service tokens (owner prefers one rotation), raw-HTTP backend rewrite (would duplicate the agentic tool loop).
